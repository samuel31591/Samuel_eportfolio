Network And Information Security Management

This module provides the background knowledge in security concepts and terminology, illustrated by hands on network technology., providing knowledge in the basics of both information security and network technology. Using network technology to illustrate some very important security concepts such as vulnerability assessments, penetration testing, forensic analysis and breach management. The module will be concluded with a look at some of the challenges that face the Internet, and a consideration of how recent and emerging network technology can be used to mitigate some of the issues presented.

    Compromising a Medical Mannequin. It has been observed that the pace of technological advancement in the medical industry is faster than the pace of cyber security considerations (Glisson et al., 2015). The critical observation is that a breach of a training tool would result in a clinician receiving incorrect guidance when problem-solving is insightful. A discrepancy between technology and security considerations in this industry can potentially result in a fatality. The studies were conducted on a mannequin model (istan), a wireless patient simulator used at the university's College of Nursing. The device can bleed, secrete bodily fluids, has blood pressure and heart rate, and breathes realistically. The simulator links with iStan software, which controls the mannequin remotely (Still, 2015). By identifying the network security solution and network protocol as the vulnerable components, brute-force attacks against the router PIN, as well as denial-of-service attacks using open-source tools such as Backtrack The tools implemented in this experiment were not sophisticated or difficult to acquire. However, this alone highlights the lack of security currently being implemented in these devices. A brute force attack that targeted the Wi-Fi and a denial-of-service attack rendering the unit inoperable were used to demonstrate the vulnerability of the medical mannequin. A Denial of service (DOS) attack is designed to shut down or make the target unusable. This is accomplished by flooding the target with a large amount of data that the device cannot handle, causing it to become unresponsive. There are several ways to mitigate brute force attacks, such as changing passwords frequently, increasing password length, limiting login attempts, and implementing multifactor authentication. A proactive approach would be setting up alerts for brute force attacks and monitoring and stopping them before they are fully deployed (Varonis, 2021). DOS attacks can be prevented by proactively monitoring a system for any unusual activity by creating a baseline. In addition to this monitoring, an effective method could also be to implement a VPN with MFA required to access the medical device (Phoenixnapp,2018). Over the years, medical devices have gone from being utterly non-connected to being both wirelessly and software controlled. It's worth bearing in mind that there is an international standard to which medical devices must certify; The IEC 62304 Medical device software, which was put in place to prevent cyber-criminal events on medical devices. However, the standard has not been updated to consider and include either network connectivity or cybersecurity (Williams, Woolward, 2015). When medical device manufacturers are required by law to implement strict security measures into the lifecycle of their products, we will likely see the threats above, and vulnerabilities decrease.

    By disabling WPS, the mannequin can't easily connect to the network, making it more vulnerable. The WiFi Protected Setup (WPS) is a wireless network security standard that makes connections between a router and wireless devices (mannequins) faster and easier. Wireless Protected Setup is a key exchange mechanism rather than a security mechanism. It facilitates connections between devices and access points and is one of the vulnerable points of a network. WPS works with WPA/WPA2/PSK and allows devices to establish connections to WLANs. Utilizing the following methods PIN: Entering a PIN on the device Push-button: A physical or virtual push-button should be pressed on both the access point and the client device during the connection phase. NFC: To use Near Field Communication, the client must bring their device close to the access point. USB drive: Necessary connection information exchange between an access point and a device is done using a USB drive. The short length of WPS PINs and the fact that both PIN parts are validated separately means that an online brute force attack on the PIN can be made within several hours, allowing an attacker to access a WLAN (mannequin device). Wireless LAN clients can authenticate their users and ensure user confidentiality through WPA or WPA2. WPA is vulnerable to the offline dictionary attack since the information required to create and verify a session key is broadcasted (Noureddine Boudriga, 2019, p.270) Using WPA2/enterprise security and mutual client and server authentication provides the most secure and protected wireless network compared to the WPA and WEP.

      AUTHENTICATION AND AUTHORIZATION; AN ESSENTIAL FOR NETWORK AND SOFTWARE SECURITY.

      Security is a crucial part of any application or environment (Bullock and Benford, 1999) and granting access to these secured applications and the environment needs to be controlled; who gets access to what? What are the rules? How is it tracked? These are managed through an access control system providing access to restricted areas of the business (SailPoint, 2021).” Access control is governed according to the space within which subjects and objects reside, and the ability to traverse space to get close to an object” by Bullock and Benford, 1999. Who(subject) gets access to resources(object) is validated through authentication and authorisation processes.

      Authentication, verifying who someone is, has long passed username and passwords to more advanced Artificial intelligence and deep learning technologies. The application of multi-factor authentication verifying an individual identity based on two to three different parameters, something you (have, know and are) works based on artificial intelligence and deep learning, where data, logs, transactions, and real-time communications are analysed to detect an intruder (Okta, 2018).

      Hardware embedded chips are evolving technologies that authenticate users using multiple levels and methods. An example is the sixth generation vPro Chips by intel (IFF Lab, 2019).

      Authorisation is to which asset one has permission to access through settings supported by security teams (Okta, 2018). The more often used to perform these authorisation processes include Role-Based Access Control, Discretionary based access control, Attribute-based access control, Rule-Based Access Control. These strategies help application developers control the access to the resources depending on the applied method and to what level the user is permitted.

      Authentication and authorisation are vital in access control management, verifying a user’s identity effectively, thus allowing authenticated users to access protected resources depending on the given authority level; these ensure that security is maintained both in the network and the software to be accessed (Wang, Bo and Zhang, 2013).

    The importance of control mechanisms (how, who, and to what extent) access is granted into a network cannot be overemphasised, authenticating a user into the network, as discussed during this period. One of the weakest links in security is the user; as pointed out by Richard (Tolofari and Meadows, 2021), users need to be educated on how to create their passwords, not using conventional passwords like 12345. ARP spoofing, also known as ARP poisoning, is a network attack in which a malicious user
sends a
forged ARP (Address Resolution Protocol) message over a local area network. This causes
the
attacker’s Media Access Control (MAC) address to be linked to the IP address of a
legitimate
computer or server on the network (Dacier et al., 2017).
In cybersecurity, there is two mitigation strategy, passive and active mitigation (Daimi
and
Peoples, 2021).
ARP monitoring is a passive mitigation strategy, monitoring or scanning to sniff packets
on
the
Queen’s medical server to detect when ARP is spoofed. When an ARP packet is received,
two
things
must be checked; if they do not match, the server is vulnerable to an ARP spoofing
attack.

identify the spoofed ARP packets inside the network. The two things are

address.
